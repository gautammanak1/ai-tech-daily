## Product & Technology Deep Dive

Weaviate's architecture represents a fundamental rethinking of database technology for the AI era. Unlike traditional databases that were designed for structured data and later adapted to handle vectors, Weaviate was built from the ground up as an AI-native vector database.

**Core Architecture:**

At its foundation, Weaviate stores data as objects with associated vector embeddings. These embeddings are AI-generated representations that capture the semantic meaning of documents, images, customers, products, and other entities. By storing both the raw object data and its vector representation together, Weaviate enables powerful hybrid search capabilities that combine the semantic understanding of vector search with the precision of keyword filtering.

**Key Technical Features:**

1. **Built-in Vector and Hybrid Search**: Weaviate supports pure vector similarity search, semantic search using natural language queries, and hybrid search that combines vector similarity with keyword matching. The hybrid search uses an alpha parameter (0-1) to control the balance between vector and keyword components, giving developers fine-grained control over search relevance.

2. **Automatic Embedding Management**: The platform can automatically handle embeddings, ranking, and auto-scaling, allowing developers to focus on shipping features rather than managing infrastructure. Weaviate supports integration with popular embedding providers and can generate embeddings on-the-fly as data is ingested.

3. **Billion-Scale Architecture**: Designed to adapt to any workload, Weaviate scales seamlessly both vertically and horizontally while optimizing costs. The architecture supports everything from small prototypes to enterprise deployments with hundreds of millions of vectors.

4. **Multi-Model Support**: Weaviate is language-agnostic by design, offering SDKs for Python, Go, TypeScript, and JavaScript, along with GraphQL and REST API access. This flexibility allows teams to use Weaviate with their existing technology stack without being locked into a specific programming language.

5. **Enterprise-Grade Features**: Production deployments benefit from role-based access control (RBAC), SOC 2 compliance, HIPAA readiness, and comprehensive security features. The platform can run securely in Weaviate Cloud or deployed on-premises in customer infrastructure.

**The Agent Skills Innovation:**

The February 2026 launch of Agent Skills represents Weaviate's strategic expansion into the agentic AI space. Agent Skills provides coding agents with standardized tools to interact with Weaviate databases more accurately and efficiently. This addresses a critical pain point in AI development: while agents like Claude Code, Cursor, and GitHub Copilot have become powerful coding assistants, they often struggle with database-specific operations without proper tooling. ## Getting Started — Code Examples

Let's dive into practical examples of how to use Weaviate in your projects. These examples demonstrate the platform's flexibility across different use cases.

### Example 1: Basic Setup and Query Operations

This example shows how to get started with Weaviate, including setting up a client and performing different types of searches:

```python
import weaviate

# Connect to Weaviate Cloud or local instance
client = weaviate.connect_to_weaviate_cloud(
    cluster_url="your-cluster-url.weaviate.cloud",
    auth_credentials=weaviate.AuthApiKey("your-api-key")
)

# Select a collection
collection = client.collections.get("SupportTickets")

# Pure vector search using embedding vectors
response = collection.query.near_vector(
    near_vector=[0.1, 0.1, 0.1],
    limit=5
)

# Semantic search using natural language
response = collection.query.near_text(
    query="login issues after OS upgrade",
    limit=5
)

# Hybrid search combining vector and keyword search
# alpha=0.75 means 75% vector, 25% keyword
response = collection.query.hybrid(
    query="login issues after OS upgrade",
    alpha=0.75,
    limit=5
)

# Process results
for result in response.objects:
    print(f"Ticket: {result.properties['title']}")
    print(f"Similarity: {result.metadata.distance}")
```

### Example 2: Creating a Collection with Automatic Embeddings

This example demonstrates how to create a collection with automatic vectorization:

```python
import weaviate
from weaviate.classes.config import Configure, Property, DataType

client = weaviate.connect_to_local()

# Create a collection with automatic embeddings
client.collections.create(
    name="Products",
    properties=[
        Property(name="name", data_type=DataType.TEXT),
        Property(name="description", data_type=DataType.TEXT),
        Property(name="price", data_type=DataType.NUMBER),
        Property(name="category", data_type=DataType.TEXT),
    ],
    # Configure vectorization with OpenAI embeddings
    vectorizer_config=Configure.Vectorizer.text2vec_openai(
        model="text-embedding-3-small",
        dimensions=1536
    )
)

# Add data - vectors are generated automatically
products = client.collections.get("Products")

with products.batch.dynamic() as batch:
    batch.add_object({
        "name": "Wireless Headphones",
        "description": "Premium noise-cancelling headphones with 30-hour battery life",
        "price": 299.99,
        "category": "Electronics"
    })
    batch.add_object({
        "name": "Running Shoes",
        "description": "Lightweight running shoes with advanced cushioning technology",
        "price": 149.99,
        "category": "Sports"
    })

# Semantic product search
results = products.query.near_text(
    query="comfortable audio equipment for travel",
    limit=3
)

for product in results.objects:
    print(f"{product.properties['name']}: ${product.properties['price']}")
```

### Example 3: Advanced Filtering with Hybrid Search

This example shows Weaviate's powerful filtering capabilities combined with hybrid search:

```python
import weaviate

client = weaviate.connect_to_local()
collection = client.collections.get("Documents")

# Hybrid search with complex filters
response = collection.query.hybrid(
    query="machine learning optimization techniques",
    alpha=0.7,  # Weight toward vector similarity
    limit=10,
    # Add structured filters
    filters=weaviate.classes.query.Filter.by_property("publish_date").greater_than(2024) & \
            weaviate.classes.query.Filter.by_property("category").equal("AI") & \
            weaviate.classes.query.Filter.by_property("word_count").greater_than(1000),
    # Return specific properties only
    return_properties=["title", "author", "publish_date", "summary"]
)

# Group by category and get aggregate information
response = collection.query.hybrid(
    query="deep learning frameworks comparison",
    limit=5,
    group_by="category",
    return_metadata=weaviate.classes.query.MetadataQuery(distance=True)
)

for group in response.groups:
    print(f"Category: {group}")
    for result in group.objects:
        print(f"  - {result.properties['title']} (distance: {result.metadata.distance:.4f})")
```
